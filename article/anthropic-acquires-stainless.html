อัปเดตใหม่ Claude Managed Agents: ควบคุมแซนด์บ็อกซ์เองและเชื่อมต่อ MCP ส่วนตัวได้แล้ว

2026-05-21 · ~1 นาที · Industry

วันนี้เรามีข่าวดีสำหรับทุกคนที่ใช้งาน Claude Managed Agents บนแพลตฟอร์ม Claude ไม่ว่าคุณจะเป็นนักพัฒนา ผู้จัดการระบบ หรือฝ่ายความปลอดภัยขององค์กร เพราะว่าตอนนี้ Claude Managed Agents สามารถทำงานในสภาพแวดล้อมแซนด์บ็อกซ์ที่คุณควบคุมเอง และเชื่อมต่อกับเซิร์ฟเวอร์ Model Context Protocol (MCP) ส่วนตัวของคุณได้แล้ว

ทำไมต้องเปลี่ยน?

ก่อนหน้านี้ เมื่อ Agent ทำงานกับเครื่องมือต่างๆ ทุกอย่างจะเกิดขึ้นบนโครงสร้างพื้นฐานของ Anthropic ซึ่งอาจไม่เหมาะสำหรับองค์กรที่ต้องการควบคุมข้อมูลที่ละเอียดอ่อน หรือมีข้อกำหนดด้านความปลอดภัยที่เข้มงวด แต่ตอนนี้ ทั้งสภาพแวดล้อมแซนด์บ็อกซ์ที่ Agent ดำเนินการเครื่องมือต่างๆ และบริการที่เชื่อมต่อ จะทำงานภายในขอบเขตขององค์กรคุณ ภายใต้การควบคุมด้านความปลอดภัยและรันไทม์ของคุณเอง

Self-hosted Sandboxes: ควบคุดสภาพแวดล้อมเอง

ด้วย self-hosted sandboxes คุณสามารถเก็บไฟล์ แพ็คเกจ และบริการที่ละเอียดอ่อนไว้ในโครงสร้างพื้นฐานของคุณเอง หรือใช้ผู้ให้บริการแซนด์บ็อกซ์ที่จัดการได้ กระบวนการทำงานของ Agent ที่จัดการการประสานงาน การจัดการบริบท และการกู้คืนข้อผิดพลาดจะยังคงอยู่บนโครงสร้างพื้นฐานของ Anthropic แต่การดำเนินการเครื่องมือจะย้ายไปที่สภาพแวดล้อมที่คุณกำหนดค่า ภายในขอบเขตของคุณ นโยบายเครือข่าย การบันทึกการตรวจสอบ และเครื่องมือด้านความปลอดภัยต่างๆ ถูกติดตั้งอยู่แล้ว และไฟล์และที่เก็บข้อมูลจะไม่ถูกส่งออกไปนอกองค์กร นอกจากนี้ คุณยังควบคุมการประมวลผลได้อย่างเต็มที่: ขนาดทรัพยากรและอิมเมจรันไทม์ถูกตั้งค่าที่ฝั่งคุณ ทำให้ Agent ที่ทำงานหนักๆ เช่น การคอมไพล์นานๆ หรือการสร้างภาพ จะได้รับ CPU, หน่วยความจำ และความสามารถที่งานต้องการ

เลือกผู้ให้บริการแซนด์บ็อกซ์ได้ตามต้องการ

คุณสามารถนำแซนด์บ็อกซ์ไคลเอนต์ใดก็ได้ที่คุณต้องการมาใช้ หรือเริ่มต้นด้วยหนึ่งในผู้ให้บริการที่เรารองรับ:

ผู้ให้บริการคุณสมบัติเด่นกรณีการใช้งาน
Cloudflareใช้ microVMs และ isolates ที่เบา ควบคุมคำร้องเครือข่ายขาออกการเชื่อมต่อบริการภายในผ่านเครือข่าย Cloudflare
Daytonaคอมพิวเตอร์แบบเต็มที่ ทำงานนานๆ และเก็บสถานะได้Agent ที่ต้องการทำงานหลายชั่วโมงและเก็บสถานะ
Modalแพลตฟอร์มคลาวด์สำหรับงาน AI สามารถใช้ทรัพยากร CPU และ GPU ได้ระบบ AI ในระดับการผลิตที่ต้องการการปรับขนาดสูง
Vercelความปลอดภัยระดับ VM, VPC peering, และการนำคลาวด์ของคุณเองมาใช้การใช้งานบนโครงสร้างพื้นฐานคลาวด์ที่มีอยู่แล้ว

MCP Tunnels: เชื่อมต่อบริการภายในองค์กร

ด้วย MCP tunnels ใหม่ Agent ของคุณสามารถเข้าถึงเซิร์ฟเวอร์ MCP ภายในเครือข่ายส่วนตัวของคุณได้ โดยไม่ต้องเปิดเผยต่ออินเทอร์เน็ตสาธารณะ ฐานข้อมูลภายใน API ส่วนตัว ฐานความรู้ และระบบติ๊กเก็ตต่างๆ จะกลายเป็นเครื่องมือที่ Agent ของคุณสามารถเรียกใช้ได้ เกตเวย์ที่คุณปรับใช้จะสร้างการเชื่อมต่อขาออกเพียงอันเดียว ไม่ต้องมีกฎไฟร์วอลล์ขาเข้า ไม่มี endpoint สาธารณะ และการจราจรถูกเข้ารหัสตลอดทาง MCP tunnels ได้รับการสนับสนุนใน Managed Agents และ Messages API และสามารถจัดการได้จากการตั้งค่าพื้นที่ทำงานภายใน Claude Console โดยผู้ดูแลระบบขององค์กร

การเริ่มต้นใช้งาน

ทั้ง self-hosted sandboxes และ MCP tunnels ทำงานภายในพื้นฐานเดิมเดียวกันที่รองรับโดย Managed Agents Self-hosted sandboxes พร้อมใช้งานในเวอร์ชันเบต้าสาธารณะ และ MCP tunnels อยู่ในระหว่างการทดสอบเบื้องต้น (research preview) หากต้องการเริ่มต้นใช้งาน MCP tunnels สามารถขอสิทธิ์การเข้าถึงได้ คุณสามารถสำรวจเอกสารประกอบของเราเพื่อเรียนรู้เพิ่มเติม ทำตามคู่มือของเราเพื่อตั้งค่าผู้ให้บริการแซนด์บ็อกซ์ หรือปรับใช้ Agent ตัวแรกของคุณใน Claude Console ได้เลย

ประสบการณ์จากผู้ใช้งานจริง

ทีมงานจากบริษัทต่างๆ ได้แสดงความคิดเห็นเกี่ยวกับการใช้งาน Claude Managed Agents ใหม่นี้: Ryan Chang จากทีม AI Engineering กล่าวว่า "Claude Managed Agents ช่วยให้เราจำลองพลังของ Agent ในเครื่อง (local agent) ได้ ในขณะที่ยังคงความน่าเชื่อถือ การจัดเวอร์ชัน และการดำเนินการเบื้องหลังของ Agent บนคลาวด์" Strib Walker หัวหน้าฝ่ายผลิตภัณฑ์กล่าวว่า "Claude Managed Agents จัดการวงจร Agent ให้เรา ในขณะที่แซนด์บ็อกซ์ของ Vercel มอบสภาพแวดล้อมที่เราสามารถปรับแต่งให้เหมาะกับงานของเราได้" ไม่ว่าคุณจะกำลังสร้าง Agent สำหรับการออกแบบ การวิเคราะห์ทางการเงิน หรือระบบอัตโนมัติใดๆ ก็ตาม การอัปเดตนี้จะช่วยให้คุณมีความยืดหยุ่นและความปลอดภัยมากขึ้นในการพัฒนาและการใช้งาน Agent บนแพลตฟอร์ม Claude แหล่งอ้างอิง: https://claude.com/blog/claude-managed-agents-updates
