AI Coding Has a "New Competitor" That Beats Claude Code, and "New Skills" That Make Claude 10x Better — 2 Technologies Developers Must Know

2026-05-11 · ~5 min · AI Agents

Introduction

If you're someone who uses Claude Code to help write code, you might have encountered the problem that when projects get larger, the AI starts to "forget" previously written code, or summarizes old information in a way that "loses details" — but now there's a new architecture called LCM (Lossless Context Management) specifically designed to solve this problem, and it has been tested to beat Claude Code at all conversation lengths, from 32K to 1 million tokens! Paired with the Skills Framework from Matt Pocock (53,900 stars on GitHub) that transforms Claude Code from an "ordinary assistant" to a "specialized expert." Today, we'll dive deep into these two technologies that will forever change how you use AI to write code.

Technology 1: LCM — The New Architecture That Beats Claude Code

The Old Problem: AI "Forgets" During Long Conversations

Imagine you have an AI write a large project with 50 files, and you've been talking for 3 hours — by the end, the AI starts getting confused, not knowing which code has already been written, what variable names were used, or what decisions were made earlier. This is what's known as "Context Management" — managing the information that the AI needs to remember in its head (context window), which has limitations. Claude Code uses a method called "Compaction" — summarizing old conversations to make them shorter when approaching the limit, but this method has the disadvantage of losing details (lossy) — like reading a 100-page book and summarizing it in just 5 pages; small but important details often get lost.

The New Solution: LCM (Lossless Context Management)

LCM is an architecture developed by Clint Ehrlich and Theodore Blackman (February 2026) that solves this problem with a completely different approach.

For visualization: If Claude Code is like someone who reads a book and writes a brief summary, LCM is like someone who creates an "intelligent index" — it doesn't summarize information, but knows where each piece of information is located and can retrieve it immediately when needed without losing anything (Lossless)

The 2 Principles That Make LCM Special:

1. Recursive Context Compression (Lossless Compression)

LCM uses a DAG (Directed Acyclic Graph) structure or "hierarchical graph" to summarize old information, but what makes it different from the old method is that it still keeps pointers to all the original information — like creating a "multi-level index" that can always go back to see the actual data. 

↓
Level 1 Summary (paragraph)
↓
Level 2 Summary (page)
↓
Level 3 Summary (chapter)
↓
AI reads only the top-level summaries → but when details are needed
→ follows pointers down to retrieve actual data immediately

2. Recursive Task Partitioning (Dividing Work into Parts)

Instead of letting AI control its own work loop (which might run indefinitely), LCM uses a system of engine-managed parallel primitives like LLM-Map — similar to changing from using GOTO to using for and while in programming. The result is that AI is guaranteed to finish its work (termination guarantee) and doesn't waste time doing redundant work (zero-cost continuity).

Results: Beats Claude Code at Every Level!

When LCM was applied to the Opus 4.6 model to create a coding agent called Volt, test results on OOLONG (a benchmark for long-context) showed it beating Claude Code at every level:

Context LengthLCM (Volt)Claude Code
32K tokensWinLose
128K tokensWinLose
512K tokensWinLose
1M tokensWinLose

Short summary: LCM isn't a new model, but a "new way of thinking" about context management — not summarizing with loss, but creating a multi-level index that can always go back to see the actual data.

Technology 2: Skills Framework — Giving Claude Code "Expertise" Like Humans

The Problem: Claude Code is Good but Not "Specialized"

Claude Code is like a top student who knows everything but isn't "specialized" in any particular subject — it's good at writing TypeScript code, for example, but doesn't know your team's conventions, testing preferences, or deployment methods.

Matt Pocock (one of the most famous TypeScript experts in the world) solved this problem by creating the Skills Framework for Claude Code — and it has gained popularity with 53,900 stars on GitHub!

What are Skills?

Skills are specialized instruction files that tell Claude Code "when doing this type of work, follow this method." It's like hiring a new employee and giving them an "operations manual" — not just saying "go write code," but also saying "here we write code this way, test this way, name variables this way."

Example Skill:

skills/
typescript.md ← Team's TypeScript writing method
testing.md ← Testing method
deployment.md ← Deployment method
code-review.md ← Code review method
CLAUDE.md ← Main instructions
CONTEXT.md ← Additional information

Why 53,900 Stars?

The Skills Framework solves problems that every developer encounters:

  1. Makes AI understand team context — no need to explain repeatedly
  1. Collects best practices — great techniques from world-class experts
  1. Customizable — each team can create their own skills
  1. Shareable — has a .claude-plugin system for installing skills from others
  1. Has verification system — scripts for verifying and listing available skills

For visualization: If regular Claude Code is like hiring a new student, Claude Code + Skills is like hiring a new student who has already undergone 3 months of specialized training — works faster, more accurately, and requires less training.

How Can You Use Them?

For Skills Framework:

Available to use today! Install from GitHub: github.com/mattpocock/skills Start by looking at existing skills → adapt them to your team Create a .claude/ directory in your project → add skill files one by one Use with CLAUDE.md to create a comprehensive instruction system

Summary

These two technologies, despite their different approaches, tell the same story: AI Coding is evolving from a "general assistant" to a "specialized intelligent system" — LCM solves the problem of AI forgetting when conversations get long by creating multi-layered lossless context tables (and has been tested to outperform Claude Code in all benchmarks!), while Skills Framework solves the problem of AI not understanding team context by providing AI with "specialized manuals" to follow — both are crucial steps that will make AI-assisted coding many times more effective. Developers who start using them today will gain a significant advantage
